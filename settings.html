Settings Manual

General Settings

This section covers the general settings of the HR1 application. You can modify various aspects of the application here.

  • Logo: You can upload your company's logo. The recommended size is 150x50 pixels. Supported formats are PNG and JPG.
  • Company Name: Enter the official name of your company. This name will be displayed in the application's header and reports.
  • Default Language: Set the default language for the application. Available languages are English, Spanish, and French.
  • Date Format: Choose the date format that suits your region (e.g., MM/DD/YYYY, DD/MM/YYYY, YYYY-MM-DD).
  • Time Zone: Select your company's time zone. This is important for scheduling and time-sensitive features.

User Management Settings

This section deals with settings related to user accounts and their roles.

  • Roles: You can create new roles and modify existing ones. Roles define the permissions and access levels of users.
  • User Status: You can activate or deactivate user accounts. Deactivated accounts cannot log in to the application.
  • Password Policy: Configure password complexity requirements, such as minimum length, required characters, and expiration settings.
  • Default Role: Set the default role assigned to new users upon registration.

Notification Settings

Configure email and in-app notification settings.

  • Email Notifications: Enable or disable email notifications for various events, such as new applications, interview schedules, and new user registrations.
  • In-App Notifications: Customize in-app notification preferences.
  • Notification Templates: Modify the content of email and in-app notification templates. You can use placeholders to include dynamic information.

Application Tracking Settings

Customize the application tracking process.

  • Application Stages: Add, edit, or delete application stages to match your hiring process. You can rename the default stages (e.g., "Applied," "Screened," "Interview," "Offer," "Hired") or create new ones.
  • Custom Fields: Create custom fields to capture additional information on application forms.
  • Hiring Teams: Create and manage hiring teams, and assign them to specific job postings.

Job Posting Settings

Configure settings related to job postings.

  • Default Job Location: Set a default location for job postings.
  • Posting Duration: Set the default duration for job postings (e.g., 30 days, 60 days).
  • Application Email: Specify the email address where applications should be sent.
